1. Introduction
This project presents a multimodal machine learning pipeline that combines facial recognition, voice verification, and tabular customer behavior modeling to secure access to product recommendations. The central idea is that predictive output should not be shown until identity is validated through two checkpoints. This makes the recommendation stage both useful and controlled.

The implemented workflow integrates three modalities: customer profile and transaction records, face images, and voice recordings. The final system demonstrates both positive authorization behavior and safe denial behavior when authentication fails.

2. Project Objectives
The project objectives were aligned to implementation quality and rubric outcomes:
- Merge customer social profile and transaction datasets into a validated engineered dataset.
- Build image and audio preprocessing pipelines with augmentation and feature extraction.
- Implement three models: facial recognition, voice verification, and product recommendation.
- Evaluate each model with Accuracy, weighted F1-score, and Loss.
- Apply multimodal decision logic that releases recommendation output only after successful identity checks.
- Demonstrate authorized and unauthorized flows using a command-line simulation.

3. System Architecture and Decision Workflow
3.1 End-to-End Operational Flow
The runtime flow is sequential. A user first submits a face image and identity is predicted by the face model. If the face is unknown or confidence is below threshold, access is denied immediately. If face validation passes, recommendation processing is unlocked. The user then submits a voice sample for second-factor verification. Recommendation output is released only when voice identity matches and passes threshold conditions.

3.2 Security and Reliability Rationale
This architecture reduces single-modality risk by requiring two independent biometric confirmations before decision release. It also improves interpretability because failure points are explicit and auditable.

4. Data Preparation, Cleaning, and Merge Validation
4.1 Data Sources and Folder Standardization
Two structured datasets were used as tabular inputs: customer_social_profiles.csv and customer_transactions.csv. Biometric inputs were organized in member-based folders for both images and audio, with unauthorized folders used for negative-path simulation. A consistent project structure across raw, processed, artifacts, and reports directories improved reproducibility and reduced path-related execution errors.

6. Image and Audio Processing with Feature Engineering
6.1 Image Collection, Augmentation, and Features
Each member contributed three facial expressions: neutral, smile, and surprised. Image augmentation included rotation, horizontal flip, and grayscale conversion to increase robustness to orientation and appearance variation. Feature extraction combined histogram descriptors and HOG descriptors, and outputs were stored in image_features.csv.

6.2 Audio Collection, Quality Review, Augmentation, and Features
Each member recorded at least two required phrases: "Yes, approve" and "Confirm transaction." Audio quality was reviewed through waveform and spectrogram plots. Augmentation included additive noise, time stretching, and pitch shifting. Features were extracted using MFCC statistics, spectral roll-off, and energy-related descriptors, and stored in audio_features.csv.

7. Model Implementation
7.1 Facial Recognition Model
The facial model was trained on engineered image features to classify identity labels. Tree-based classification was chosen because of strong performance on tabularized handcrafted feature vectors and minimal preprocessing overhead.

7.2 Voice Verification Model
The voice model was trained on engineered audio features to classify identity. This model serves as the second-factor verification component in runtime decision logic.

7.3 Product Recommendation Model
The recommendation model was trained on merged customer behavior features with multimodal alignment where available. Pipeline preprocessing handled mixed numeric and categorical fields for consistent inference behavior.

7.4 Training Reproducibility and Artifact Persistence
Training used deterministic split settings and serialized model outputs for reproducible reuse. Saved artifacts were consumed directly by CLI simulation.

8. Evaluation Results and Multimodal Logic Performance
8.1 Evaluation Metrics
All models were evaluated using Accuracy, weighted F1-score, and Loss to capture correctness, class-aware behavior, and confidence-sensitive performance quality.

8.2 Final Results
Current recorded metrics are:
- Facial Recognition: Accuracy 0.8750, F1-weighted 0.8250, Loss 0.4302
- Voice Verification: Accuracy 0.8333, F1-weighted 0.7667, Loss 0.3759
- Product Recommendation: Accuracy 0.7955, F1-weighted 0.7878, Loss 0.7998

8.3 Interpretation
Identity models achieved strong performance for the available sample scale, while recommendation performance is practical for the current feature space. The multimodal gate increases trustworthiness by requiring agreement across two biometric checks prior to output release.

9. System Demonstration and Deliverable Evidence
9.1 Authorized and Unauthorized Simulation
The CLI simulation demonstrated both valid and invalid paths. In the authorized path, matching face and voice produced recommendation approval. In the unauthorized path, unknown or mismatched biometric input caused access denial.

9.2 Produced Artifacts
The final output package includes merged_features.csv, image_features.csv, audio_features.csv, trained model artifacts, metrics JSON, and visual report assets. These files provide direct reproducibility and evidence for assessment.

9.3 Limitations and Improvement Plan
Current limitations include small biometric sample size, single split evaluation, and limited anti-spoofing hardening. Recommended improvements include larger and more diverse data, cross-validation, calibration experiments, and deep embedding comparisons for face and voice.
